# Review Pulse — AI-Powered Product Intelligence Platform

> Turns raw App Store and Play Store reviews into validated, prioritized weekly intelligence for product teams.

**[Live Dashboard →](https://REPLACE-WITH-YOUR-RAILWAY-URL.up.railway.app)**

---

## What This Does

Review Pulse automatically ingests hundreds of public app reviews, clusters them into themes using ML, validates every quote and claim through a multi-stage pipeline, and produces a weekly intelligence report with confidence-scored, prioritized insights — not just summaries.

**Built for:** Product teams, Support leads, and Leadership at consumer fintech companies.
**Current coverage:** INDMoney, Groww (expandable via config).

## The Pipeline

```
App Store + Play Store Reviews (8–12 week rolling window)
        │
        ▼
   PII Scrubbing
        │
        ▼
  Sentence Embeddings (BGE-small, local)
        │
        ▼
  UMAP → HDBSCAN Clustering
        │
        ▼
  LLM Theme Summarization (Groq, llama-3.3-70b)
        │
        ▼
  5-Stage Quote & Theme Validation
  ├── RapidFuzz exact-match (≥90)
  ├── LLM relevance scoring (≥2/3)
  ├── Theme summary validation
  ├── Hallucination culling (drop unsupported themes)
  └── Confidence scoring (0.6×volume + 0.4×validation)
        │
        ▼
  Priority Scoring + Trend Detection
        │
        ▼
  Dashboard + Google Docs + Gmail Draft
```

## Key Design Decisions (the PM thinking)

**Why 5-stage validation?**
V1 used raw LLM quotes — many were hallucinated. V2 added fuzzy matching — quotes were real but irrelevant to their theme. V3 added semantic relevance scoring. V4 caught hallucinated *summaries* (e.g. "gaming experience" in a fintech report). V5 drops themes entirely if zero quotes survive validation. Each stage was a response to a real failure mode found in testing.

**Why confidence and priority scores?**
A theme mentioned by 200 people with 80% quote validation is fundamentally different from one mentioned by 5 people with 20% validation. Priority = f(volume, severity, confidence) — so leadership sees what matters most, not just what's loudest.

**Why theme persistence tracking?**
Individual weekly reports answer "what happened this week." The persistence heatmap answers the harder question: "which problems aren't going away?" That's the insight that changes a roadmap.

**Why validated quotes matter?**
Every quote in every report traces back to a real, scrubbed review. If a stakeholder asks "did a customer actually say this?" — the answer is always provably yes.

## How to Re-Run for a New Week

```bash
# 1. Clone and set up
git clone https://github.com/Flukeshotz/PULSE.git
cd PULSE
python3 -m venv venv && source venv/bin/activate
pip install -r requirements.txt

# 2. Set your Groq API key
export GROQ_API_KEY="your_key_here"

# 3. Run the pipeline for a product
python3 src/cli.py --product indmoney          # current week
python3 src/cli.py --product indmoney --week 2026-W25  # specific week
python3 src/cli.py --product groww              # different product

# 4. Reports are saved to data/reports/{product}/{iso_week}.json
# 5. Copy to frontend and rebuild
cp -R data/reports frontend/public/data
cd frontend && npm run build
```

Each run is **idempotent** — re-running the same product + week won't duplicate data.

## Theme Legend

Themes are auto-generated by the clustering + LLM pipeline, but common recurring categories include:

| Category | Examples |
|---|---|
| **Trading & Execution** | Order execution delays, brokerage charges, transaction failures |
| **App Quality** | Crashes, performance, UI bugs, compatibility issues |
| **Trust & Transparency** | Hidden charges, misleading information, account security |
| **Support** | Response times, resolution quality, escalation gaps |
| **Features & UX** | Missing features, navigation confusion, onboarding friction |

## Tech Stack

| Layer | Technology |
|---|---|
| Ingestion | Python, google-play-scraper, iTunes RSS |
| Embeddings | sentence-transformers (BGE-small, local — zero API cost) |
| Clustering | UMAP + HDBSCAN |
| LLM | Groq free tier (llama-3.3-70b-versatile) |
| Validation | RapidFuzz + LLM semantic scoring |
| Frontend | React + Vite + Tailwind + Recharts |
| Deployment | Railway (Caddy static hosting) |
| Scheduling | GitHub Actions (weekly Monday AM IST) |

**Total API cost per run: $0.** Embeddings are local, LLM is on Groq free tier.
